I started hacking my solutions to Project Euler using python but recently decided to do them in JavaScript. I could easily create a simple web page to execute my JavaScript solutions however the lazy developer in me didn't want to. I wanted to run my scripts outside of the context of a web browser.

A quick search of the interwebs yielded good results. Turns out there are a few runtimes out there for executing JavaScript from the command-line. If you are running an OSX Leopard or later you can use the JSC command-line utility. Create a symlink to the executable as follows: {% highlight bash %} $ sudo ln -s /System/Library/Frameworks/JavaScriptCore.framework/Versions/Current/Resources/jsc /bin/jsc {% endhighlight %}

I went with Mozilla's SpiderMonkey. The install was simple enough using Homebrew: {% highlight bash %} $ brew install spidermonkey {% endhighlight %} Once installation is complete you can run the interpreter interactively with the following command: {% highlight bash %} $ js {% endhighlight %} Execute a script called solution.js as follows: {% highlight bash %} $ js solution.js {% endhighlight %}

Keep in mind that you are running outside of the context of the browser so as a result functions that are available in the browser may not be available in your JavaScript runtime and vise versa.
